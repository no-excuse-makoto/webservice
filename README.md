# マイコン - 自殺を防止する相談マッチングサービス

## 概要

**マイコン** は、悩みや不安を抱えるユーザー同士が気軽に相談し合える **相談マッチングプラットフォーム** です。  
「似た経験をした人に相談したい」「身近に話せる人がいない」といったニーズに応え、  
**孤立を防ぎ、自殺を予防するゲートキーパーアプリ**として設計しました。

誰かのつらさに「共感」できるだけで、誰かの支えになれる。  
そんな **“共感の力” を可視化し、人と人をマッチングする**ことを目的としています。

---

## 開発背景

現在、日本では自殺対策として各地に**福祉窓口や電話相談窓口**が設けられていますが、  
支援の人員不足や心理的ハードルの高さから、十分に機能していない現状があります。  

さらに、これらの窓口では多くの場合、相談者の悩みに対して**「傾聴」にとどまり、悩みの根本や深い部分にまで踏み込んだアプローチが難しい**という課題があります。  
そのため、悩みを抱えた人が本当に求める支援に届かず、孤立を深めてしまうケースも少なくありません。

私はこうした社会課題に対し、**もっと気軽に、かつ悩みの深い部分にも寄り添える民間レベルの仕組みが必要だ**と感じ、このサービスを開発しました。

マイコンは、「似た経験をした人に話したい」「身近に話せる人がいない」というニーズに応え、  
匿名で気軽に投稿し、共感を得て相談相手とつながれる相談マッチングサービスです。

「共感」が人を救う――  
**共感の力で悩みの根本に寄り添い、人を支えるゲートキーパーアプリ**として、孤立や自殺の予防に貢献することを目指しています。

## 開発スケジュール

### 第一段階（基本機能）
- ユーザー認証（登録・ログイン・ログアウト・情報更新・退会）  
- 投稿（通常投稿・カテゴリー投稿・タグ投稿・匿名投稿）  
- 共感機能（共感・共感取消・共感数表示）  
- 相談相手選択機能（共感リストから選択可能）  
- 1対1チャット機能（Action Cable検討中）  
- 利用規約の実装

### 第二段階（拡張機能）
- 自動マッチング機能（フィルター条件によるマッチング）  
- 評価機能（相談後評価、信頼度の数値化、高評価ユーザーにバッジ表示）  
- 悩み掲示板機能（匿名強化、回答への評価、限定公開投稿）  
- 通知機能（新メッセージや共感の通知）  
- テストコード作成（RSpecによる単体・統合テスト）  
- UI/UXのブラッシュアップ  
- CI/CD導入（GitHub Actions）  
- Docker環境導入  
- AWSインフラ化（ECS, ECR, RDS）  
- SPA化（Vue.jsによるフロントエンド分離）

### 第三段階（高度機能）
- 音声通話・ビデオ通話機能（外部サービス連携予定）  
- カレンダー・スケジュール管理機能（相談日調整、外部カレンダー同期予定）  
- 多言語対応（予定）

## 使用技術

- **バックエンド:** Ruby on Rails  
- **フロントエンド:** HTML、CSS、Vue.js（現在学習中であり、今後SPA化のための主要技術として導入予定）  
- **データベース:** PostgreSQL  
- **環境構築:** Docker  
- **CI/CD:** GitHub Actions  
- **デプロイ:** AWS（ECS, ECR, RDS など）  
- **テスト:** RSpec  
- **バージョン管理:** Git / GitHub

---

## Vue.jsについて

現在、Vue.jsは未実装であり、これから学習を進めていく段階です。  
将来的にはフロントエンドのSPA（Single Page Application）化を目指しており、  
RailsのビューとAPIを分離し、Vue.jsを用いたより動的でユーザーフレンドリーなUIの実装を計画しています。

Vue.jsの導入により、以下のような効果を期待しています。

- ユーザー操作に対する即時レスポンスと快適なUXの実現  
- フロントエンドの状態管理やルーティングを効率的に行うことで複雑な画面遷移やリアルタイムチャット機能を実装しやすくする  
- コンポーネント化により再利用性と保守性の向上  
- 将来的な機能拡張やマルチデバイス対応の柔軟性向上  

学習と実装は並行して進め、サービスの品質向上とユーザー体験の向上に努めてまいります。
